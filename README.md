# Multiplayer-Quiz-System-Real-Time-Client-Server-Architecture-CS408-PROJECT
A multiplayer client-server quiz game application built with Python. It features graphical user interfaces (GUIs) and communicates strictly via TCP sockets.

This project is a multiplayer quiz game implemented as a client-server application using Python. The application relies entirely on TCP sockets for network communication. Instead of a console-based structure, it includes dedicated graphical user interfaces (GUIs) for both the server administrator and the connecting players (clients).  Key Features:Concurrent Multiplayer: The server module can accept and handle multiple client (player) connections simultaneously.  TCP Socket Communication: All network communication within the application is strictly performed over TCP sockets.  Speed-Based Scoring: Correct answers are awarded 1 point, while the first player to submit a correct answer receives additional bonus points equal to the number of players minus one.  Dynamic Scoreboard: A scoreboard displaying the current total scores and rankings (1st, 2nd, 3rd, etc.) of all players is updated and broadcasted to all GUIs after each question and at the end of the game.  Status and Error Handling: All operations, including disconnections, duplicate username attempts, and game events, are reported in detail through list boxes on the respective GUIs.
